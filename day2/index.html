    实习笔记 Day 2｜torch_memory_saver仓库 + vllm sleep mode

    一. torch_memory_saver

    torch_memory_saver是一个开源的高性能 CUDA 内存管理工具：

    暂停和恢复张量的cuda内存占用，保持用户的虚拟地址不变（应用程序无需修改指针引用），恢复后重新分配显存，绑定到虚拟地址上 cuda虚拟内存管理api替代cudamalloc（cudaruntime api）（被拦截）比如malloc，free替代cudamalloc，cudafree vllm的sleep_mode除了将gpu内存释放，还实现了根据tag offload到cpu： kvcache的逻辑和torch_memory_saver一样。权重是vllm自身的offload逻辑：level1 全部采用torch_memory_saver逻辑：level2

    核心机制

    暂停（Pause）：释放GPU物理内存，但保留虚拟地址 恢复（Resume）：分配新的物理内存，重新映射到原虚拟地址 优势：应用程序指针引用无需修改，避免指针重定位开销

    虚拟地址保持 vs 数据保持

    # 暂停前：虚拟地址 0x1000 → 物理内存A [数据: 100]
# 暂停后：虚拟地址 0x1000 → 无效（无物理映射）
# 恢复后：虚拟地址 0x1000 → 物理内存B [随机值]

# 关键：只保持内存位置，不保持数据内容！
# 应用层需负责数据保存/恢复

    二. sleep VS no sleep

    无sleep时，重新加载和初次加载耗时相同 sleep时，模型切换而言（权重加载）：唤醒比重新加载快20倍。推理而言（预热+缓存预热+推理）：唤醒后首次推理比加载后首次推理快（唤醒后不需要再预热，每次加载后都需要重新预热：重启python进程和cuda上下文/重新初始化内存分配器/重新捕获cuda图/重新jit编译内核，所以也称为冷启动）

    QA

    级别2为什么模型切换和首次推理都比冷启动快？：冷启动每次首次推理都包含了预热时间，级别2模型切换有保留虚拟地址的优势。 级别1首次模型加载为什么比级别2快，级别2后续唤醒不需要预热，为什么快那么多，不是应该和首次加载时间（不含预热）一样吗：区别在于保留了虚拟地址
